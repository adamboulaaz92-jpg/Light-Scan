Overview

Light-Scan is not just a port scanner — it is a complete Network Toolkit and Mini-Framework designed for security professionals, network administrators, and penetration testers. It is built to be modular, extensible, and ready for enterprise use.

At its core, Lightscan is a powerful, multi-threaded port scanner built with Python and Scapy. But it goes far beyond scanning: it includes a scripting engine (LSSE) with 14 built-in scripts for web and DNS reconnaissance, a packet sniffer, a result exporter, a GUI interface, and a packet crafting laboratory — all working together as a cohesive framework.

Lightscan — The Core Scanner

Lightscan is the heart of the entire toolkit. It is a high-performance, multi-threaded port scanner built with Python and Scapy, designed to deliver enterprise-grade network reconnaissance in a lightweight, flexible package.

Unlike traditional scanners that sacrifice accuracy for speed or vice versa, LightScan strikes a careful balance. It supports 14 distinct scan types, from the standard TCP Connect and SYN Stealth scans to advanced techniques like FDD (Firewall Detection Scan) — a proprietary scan type exclusive to Light-Scan.

Lightscan also features 12 host discovery methods, including ICMP Echo, Timestamp, Address Mask, and the rare Information (Type 15) and Solicitation (Type 10) pings — both of which are not supported by Nmap. By default, LightScan performs ICMP Echo (IPv4) and ICMPv6 Echo (IPv6) without requiring any additional flag. With 6 speed presets (from 2 to 500 threads), custom packet crafting (TTL, HLIM, source port, payload, IP ID, and flags), and a +11500 port to service mapping database, Lightscan gives you complete control over every scan.

Lightscan

v1.1.7 Core Scanner

Lightscan is the main scanning engine. It handles everything from host discovery to port scanning, service detection, OS fingerprinting, and banner grabbing. It is designed to be flexible, fast, and accurate.

Role

  • Perform host discovery using multiple methods (ARP, ICMP, TCP, UDP, etc.)
  • Scan ports using 14 different scan types (TCP, SYN, UDP, NULL, FIN, ACK, XMAS, WINDOW, MAIMON, FDD, FTP-BOUNCE, IPPROTO, PING, IDLE)
  • Detect services using a database of 11500+ port-to-service mappings
  • Grab banners from open services
  • Perform OS fingerprinting (Windows 2000 → 11, Linux, macOS, etc.)
  • Detect firewalls and filter rules
  • Customize packets with TTL, HLIM, source port, payload, IP ID, and flags

Options

-T, --target Target IP, hostname, or CIDR range
-p, --port Port(s) to scan (single, range, or list)
-st, --scan-type Scan type: TCP, SYN, UDP, NULL, FIN, ACK, XMAS, WINDOW, MAIMON, FDD, FTP-BOUNCE, IPPROTO, PING, IDLE
-s, --speed Speed preset: paranoid, slow, normal, fast, insane, Light-mode
-t, --threads Number of threads to use
-tm, --timeout Timeout in seconds
-mx, --max-retries Max number of retries if no response
-b, --banner Enable banner grabbing
-O, --os Enable OS fingerprinting

-F Scan only the top 100 ports
-sn Host discovery only (no port scanning)
-Pn, --no-ping Skip host discovery
-f, --fragmente Fragment packets for stealth
-ttl TTL IPv4 Time To Live
-hlim HLIM IPv6 Hop Limit

-payload PAYLOAD Custom payload data

LightSniff — Packet Capture Tool

LightSniff is a lightweight, feature-rich packet sniffer built for network analysis and troubleshooting. It leverages BPF (Berkeley Packet Filter) syntax for precise traffic filtering and supports both live capture and PCAP export.

LightSniff

v1.0.1 Packet Capture

LightSniff captures and analyzes network traffic in real-time. It supports multiple protocol filters, MAC address filtering, VLAN detection, and PCAP file export. It can also read existing PCAP files.

Role

  • Capture live network traffic
  • Filter traffic using BPF syntax
  • Filter by protocol (TCP, UDP, ICMP, ARP)
  • Filter by MAC address
  • Detect VLAN tags (802.1Q)
  • Export to PCAP/PCAPNG/LBN files
  • Read existing PCAP/PCAPNG/LBN files
  • Support for TLS 1.0, 1.1, 1.2, 1.3, and SSLv3

Options

-i, --interface Network interface (e.g., eth0, Wi-Fi)
-f, --filter BPF filter (e.g., 'tcp port 80', 'icmp')
-c, --count Number of packets to capture (0 = infinite)
-w, --write Save to PCAP/PCAPNG file
-r, --read Read packets from PCAP/PCAPNG file (offline mode)
--bin-save Save to LightBin binary format (.lbn)
--bin-load Load from LightBin binary format (.lbn)
-C, --compress To compress saved output (only for .lbn)
-v, --verbose Show detailed packet info
--no-promisc Disable promiscuous mode
-q, --quiet Quiet mode (no banner)
--eth Show Ethernet frame info (MAC addresses, frame type)
--vlan Show VLAN tags (802.1Q)
--arp Show only ARP packets
--tcp Show only TCP packets
--udp Show only UDP packets
--icmp Show only ICMP packets
--mac Filter by source or destination MAC address

Usage Examples

LightSniff -i eth0 Capture all traffic on interface eth0
LightSniff -i eth0 -f 'tcp port 80' -w http.pcap Capture HTTP traffic and save to PCAP
LightSniff -i Wi-Fi --tcp -c 100 -v Capture 100 TCP packets with verbose output
LightSniff -i eth0 --arp --eth Show ARP packets with Ethernet frame details
LightSniff -i eth0 --mac aa:bb:cc:dd:ee:ff Filter by MAC address
LightSniff -i eth0 --tcp --udp --icmp Capture TCP, UDP, and ICMP packets together

LightSave — Results Exporter

LightSave is a dedicated result exporter that captures the output of Lightscan command and saves it in a structured format. It supports 8 export formats and is designed for easy integration with reporting pipelines and SIEM systems.

LightSave

v1.0.1 Results Exporter

LightSave captures the output of any Lightscan command and saves it in a structured format. It supports LIGHT, TXT, HTML, XML, CSV, JSON, PDF and YAML formats, making it easy to integrate with reporting tools and SIEM systems.

Role

  • Capture output of any LightScan command
  • Export in 8 formats: LIGHT, TXT, HTML, XML, CSV, JSON, PDF, YAML
  • Automated saving with timestamps
  • Machine-readable output for automation
  • Clean, well-formatted files

Options

-C Full Lightscan command (must be enclosed in quotes)
-S {light,txt,html,xml,csv,json,pdf,yaml} Export format

Usage Examples

LightSave -C "Lightscan -T 192.168.1.1 -F -st UDP" -S xml Save UDP scan results as XML
LightSave -C "Lightscan -T 192.168.1.0/24 -s fast" -S html Save network scan as HTML report
LightSave -C "Lightscan -T scanme.nmap.org -p 22,80,443" -S json Save scan results as JSON for automation
LightSave -C "Lightscan -T 192.168.1.100 -O -b" -S pdf Save OS and banner grab results as PDF

LightLab — Packet Crafting Laboratory

LightLab is an interactive packet crafting laboratory that allows users to build custom packets from scratch using a simple command-line interface. It supports all major protocol layers and includes built-in templates for common network operations.

LightLab

v1.0.0 Packet Crafting

LightLab is an interactive packet builder that allows you to construct custom packets layer by layer. It supports ethernet, VLAN, ARP, IPv4, IPv6, TCP, UDP, ICMP, NDP, HTTP, DNS, and raw layers. You can send packets, save them to PCAP, and load existing PCAP files.

Role

  • Build packets from scratch using the new command
  • Set parameters using set command
  • View packet structure with show
  • Send packets with send (with count and verbose)
  • Save and load PCAP/PCAPNG/LBN files
  • Built-in templates for common operations

Commands

new <layer> Add a layer: ether, vlan, arp, ip, ipv6, tcp, udp, icmp, ndp_rs, ndp_ra, ndp_na, ndp_ns, icmpv6, icmpv6_echo, http, dns, raw
delete <layer> Delete a layer
params <layer> Show available parameters for a layer
set <layer>.<param>=<value> Set a parameter value
show Show current packet structure
clear Clear all layers
send [count] [-v] Send packet (count=number, -v=verbose)
timeout <seconds> Set response timeout
interval <seconds> Set interval time between packets
templates Show example configurations
save <filename.pcap/.pcapng> Save current packet to PCAP
load <filename.pcap/.pcapng> Load packet from PCAP
savebin <filename.lbn> Save current packet to LightBin
loadbin <filename.lbn> Load packet from LightBin
help Show this message
exit Quit LightLab

LSSE — Light-Scan Scripting Engine

The Light-Scan Scripting Engine (LSSE) is what makes Light-Scan a true mini-framework. With 13 built-in scripts, you can perform web and DNS reconnaissance without leaving the Light-Scan ecosystem.

LSSE

v1.0.6 Scripting Engine

LSSE is the scripting engine that extends Light-Scan's capabilities. It includes 13 built-in scripts for web and DNS reconnaissance, including spidering, HTTP header analysis, DNS zone transfers, and subdomain enumeration. Scripts can be used independently or as part of a larger workflow.

Role

  • Extend Light-Scan's capabilities with scripts
  • Perform web reconnaissance (spidering, header analysis, directory brute force)
  • Perform DNS reconnaissance (lookup, NS records, zone transfer, subdomain fuzzing)
  • Analyze security headers, cookies, and HTTP methods
  • Support for wordlists, custom extensions, and status codes
