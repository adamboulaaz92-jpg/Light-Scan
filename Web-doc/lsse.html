⚡ LSSE Light-Scan Scripting Engine v1.1.7

13 Built-in Scripts HTTP/HTTPS DNS Extend Light-Scan with powerful reconnaissance scripts

Overview

The Light-Scan Scripting Engine (LSSE) is what makes Light-Scan a true mini-framework. It extends the core scanner's capabilities with 13 built-in scripts for web and DNS reconnaissance, security analysis, and information gathering.

Each script is designed to be modular and focused, performing a single task well. Scripts can be used independently or chained together for complex workflows.

Usage

Lightscan --lsse --script <name> [required_args] [optional_args]

# Examples:
Lightscan --lsse --script http-headers --domain google.com -sp 443
Lightscan --lsse --script dns-zone-transfer --domain example.com
Lightscan --lsse --script spider --url https://scanme.nmap.org --mxp 50 --mxd 3
OptionDescription
--lsseEnable LSSE scripting engine
--script SCRIPTScript to execute
--url URLTarget URL for HTTP/HTTPS scripts
--domain DOMAINTarget domain for HTTP and DNS scripts
-sp SPPort(s) for scripts
--redirectFollow HTTP redirects
-W, --wordlistWordlist for scripts
--extensionsFile extensions for web scripts
--status-codesStatus codes for web scripts
--mxp MXPMax pages to get (spider)
--mxd MXDMax depth to crawl (spider)
--dns-serverDNS server for DNS scripts
--lsse-lstList all available scripts

HTTP / HTTPS Scripts

1. spider

safe discovery --url
Category: safe/discovery/http_https • Required: --url • Optional: --mxd, --mxp
Recursively crawls websites for links, forms, and resources. Useful for mapping web applications and discovering hidden endpoints.
Lightscan --lsse --script spider --url https://scanme.nmap.org --mxp 50 --mxd 3

2. http-robots

safe discovery --domain, -sp
Category: safe/discovery/http_https • Required: --domain, -sp • Optional: None
Fetches and parses robots.txt for hidden paths and directories that the site owner has disallowed for crawlers.
Lightscan --lsse --script http-robots --domain example.com -sp 80

3. http-cert

safe analysis --domain, -sp
Category: safe/analysis/https • Required: --domain, -sp • Optional: None
Grabs SSL/TLS certificate information including issuer, validity period, SANs, and cipher suite details. Useful for security auditing.
Lightscan --lsse --script http-cert --domain google.com -sp 443

4. script

safe discovery --url
Category: safe/discovery/http_https • Required: --url • Optional: None
Detects <script> tags in HTML pages and extracts their sources. Useful for discovering JavaScript files and tracking external scripts.
Lightscan --lsse --script script --url https://example.com

5. http-title

safe discovery --domain, -sp
Category: safe/discovery/http_https • Required: --domain, -sp • Optional: --redirect
Extracts webpage titles from HTTP responses. Useful for identifying services, applications, and default pages.
Lightscan --lsse --script http-title --domain example.com -sp 80 --redirect

6. http-dir

medium discovery --url
Category: medium/discovery/http_https • Required: --url • Optional: --wordlist, --status-codes, --extensions
Brute forces directories and files on web servers using a wordlist. Supports custom extensions and status code filtering.
Lightscan --lsse --script http-dir --url http://192.168.1.1 -W common.txt --extensions php,html,txt

7. http-headers

safe analysis --domain, -sp
Category: safe/analysis/http_https • Required: --domain, -sp • Optional: --redirect
Fetches HTTP headers and checks for missing security headers including: HSTS, CSP, X-Frame-Options, X-Content-Type-Options, Referrer-Policy, Permissions-Policy, and X-XSS-Protection.
Lightscan --lsse --script http-headers --domain google.com -sp 443

8. http-methods

safe discovery --domain, -sp
Category: safe/discovery/http_https • Required: --domain, -sp • Optional: None
Checks which HTTP methods are allowed by the server. Identifies safe methods (GET, HEAD, POST) and dangerous methods (PUT, DELETE, TRACE, OPTIONS).
Lightscan --lsse --script http-methods --domain example.com -sp 80

9. http-cookie

safe analysis --domain, -sp
Category: safe/analysis/http_https • Required: --domain, -sp • Optional: --redirect
Checks cookies for Secure and HttpOnly flags. Identifies session cookies that may be vulnerable to theft.
Lightscan --lsse --script http-cookie --domain example.com -sp 443

DNS Scripts

10. dns-lookup

safe discovery --domain
Category: safe/discovery/dns • Required: --domain • Optional: --dns-server
Fast DNS lookup for IPv4 and IPv6 addresses. Resolves domain names to IP addresses and shows all associated A and AAAA records.
Lightscan --lsse --script dns-lookup --domain google.com

11. dns-ns

safe discovery --domain
Category: safe/discovery/dns • Required: --domain • Optional: --dns-server
Gets the Name-Server (NS) records of a domain. Identifies authoritative DNS servers for the target domain.
Lightscan --lsse --script dns-ns --domain example.com

12. dns-subdomain-fuzzing

medium discovery --domain
Category: medium/discovery/dns • Required: --domain • Optional: --wordlist, --dns-server
Brute forces subdomains using a wordlist. Identifies hidden subdomains and potential attack surfaces. Useful for reconnaissance and asset discovery.
Lightscan --lsse --script dns-subdomain-fuzzing --domain example.com -W subdomains.txt

13. dns-zone-transfer

medium extracting --domain
Category: medium/extracting/dns • Required: --domain • Optional: --dns-server
Attempts an AXFR zone transfer to enumerate all DNS records for a domain. This is a classic information disclosure vulnerability when misconfigured DNS servers allow zone transfers.
Lightscan --lsse --script dns-zone-transfer --domain example.com
